<private> <static> ADF.deps.modules
Modules should be defined with ADF.addModules. An array of JS modules to be loaded. Each file path is relative to the JS directory containing this framework and the .js extension is not required. It is recommended that these files be in a subdirectory called modules. If the modules have their own JS plugin dependencies, those should be required by the modules themselves, which is explained in the module documentation. These modules are loaded synchronously, and various callbacks are available to each module so that they may run code immediately when they are ready, when the DOM is ready, or when the framework is ready after all the modules have been loaded.
<private> <static> ADF.deps.plugins
Plugins should be defined with ADF.addPlugins. An array of JS plugin files to be loaded. Each file path is relative to the JS directory containing this framework and the .js extension is not required. 
["plugins"] would load the script plugins.js if it exists in the same directory as
					this script.
