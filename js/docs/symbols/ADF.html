<private> <static> ADF.addModule()
Called in a loop after the modules defined in ADF.deps.modules have been loaded with require.js. Adds a module to the framework, storing it in ADF.modules using the module's name. If no name is defined addModule throws a warning to the console and returns. If the module has a name then addModule runs some business logic (ADF.getTileCallbacks) and fires the appropriate ready callbacks if the module has them starting first with moduleReady and then domReady if the DOM is ready.

{Module}

<static> ADF.call(api, options)
Calls a loaded module's API method with provided options. Use this in modules when you need to call API methods on other modules. It is recommended to also use this from a module to call its own API methods, since modules could eventually be unloaded or modified in a way that only the framework can track. API methods are called with the module's API object as the scope, so using the keyword "this" in API methods refers to the API object. If call returns null, the API call failed, otherwise it will return whatever the API method returns, so it is recommended to NOT return null from your API methods since a module that checks for null will assume that the call failed. Module API methods cannot be called until the module is loaded, so only make module API calls after the framework ready event has been called. 
Framework.call('coolModule.awesomeMethod', { radness: "extreme" });
Parameters:
{String} api
A string defining the the module and API method separated by a period like so: "moduleName.apiMethod". See the Module documentation for more info on defining and calling module APIs.
{Anything} options Optional
Optional argument to pass to the API method. Could be anything but it is recommended that it be an object for extensibility.

<static> ADF.publish(event, anything)
Publish a module event. If any other modules have subscribed to the event, their callback functions will be executed with the additional arguments that you supply. Note that due to the format of the first argument ("moduleName.eventName") it is technically possible to publish any module's events from any other--which is not recommended. 
			//from some module that wants to hook into an event
			Framework.subscribe('coolModule.awesomeEvent', function(one, two three) {
				//do stuff
			});
			//from coolModule calling the event
			Framework.publish('coolModule.awesomeEvent', one, two, three);
Parameters:
{String} event
A string representing the module and event being published in the format "moduleName.eventName".
{Any} anything
Each subsequent argument given to publish will be passed along to functions that are subscribed to this event.
